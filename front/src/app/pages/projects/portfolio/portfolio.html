Portfolio

Introduction

Le projet Portfolio est un mini site internet qui me permet de vous présenter mon travail. Il n'est pas vraiment un projet car n'implique pas d'autres collaborateurs. Cependant, j'ai décider de construire ce site comme s'il s'agissait d'un projet à réaliser en équipe. Dans cette page, vous pourrez voir comment la conception du projet a été réalisée et tous les documents que j'ai utilisé pour développer ce site.

Phase d'initialisation

Charte de projet

Nom du projet : Portfolio

Sponsor : Frédéric LE COAT

Chef de projet : Frédéric LE COAT

Intérêts / justifications du projet : J'ai besoin de contruire un portfolio pour me présenter et démontrer mes compétences au travers de projets concrêts.

Description du projet : Ceci est un simple site portfolio qui doit me présenter. Il listera mes compétences, mes projets et permettra de me contacter.

Elements nécessaires : Une partie présentation, une autre pour la liste de mes compétences, une qui listera tous les outils que j'utilise, une autre pour lister et détailler les projets dont j'ai participé à la réalisation et enfin une partie contact.

Critères d'acceptation : Chaque partie doit être mise en place et fonctionnelle. Tous les projets doivent être présentés.

Principaux risques : Oublie d'un projet, oublie de contacts, mauvaises redirection pour les contacts, site indisponible.

Objectif Critère de réussite Valideur
Réaliser un site portfolio complet qui permet à un visiteur de comprendre ce que je suis capable de réaliser. On attend plusieurs sections complètes dans ce projet : "présentation", "projets", "réseaux" et "compétences". Chaque partie est validée individuellement et doit proposer un contenu complet. Je serai le seul valideur de l'avancée. Je me base sur une liste exhaustive de ce que doit contenir ce genre de site et sur toutes les informations que j'ai sur mes projets et compétences.
